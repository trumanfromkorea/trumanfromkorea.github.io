## 클로저 (Closures)

클로저는 코드블럭으로 C, Objective-C 의 block 과 다른 언어의 lambdas 와 비슷합니다. 

- 클로저는 사용자의 코드 안에서 전달되어 사용할 수 있는 로직을 가진 중괄호로 구분된 코드 블럭이며, 일급 객체의 역할을 할 수 있습니다.
- 일급 객체는 전달 인자로 보낼 수 있고, 변수/상수 등으로 저장하거나 전달할 수 있으며, 함수의 리턴값이 될 수도 있습니다.
- 참조 타입입니다.
- 함수는 클로저의 한 형태로, 이름이 있는 클로저입니다.

## 클로저 표현 (Closure Expressions)

함수로 따로 정의된 형태가 아닌 인자로 들어가 있는 형태의 클로저를 인라인 클로저라고 부릅니다.

클로저 표현은 인라인 클로저를 명확하게 표현하는 방법으로 문법에 초점이 맞춰져 있습니다. 클로저 표현은 코드의 명확성과 의도를 잃지 않으면서도 문법을 축약해 사용할 수 있는 다양한 문법의 최적화 방법을 제공합니다.

### 정렬 메소드

Swift 표준 라이브러리에서 제공하는 `sorted(by:)` 메소드를 이용해보겠습니다.

`sorted(by:)` 메소드는 2개의 인자를 갖는 클로저를 인자로 사용합니다. 타입이 T 인 배열은 `(T, T) -> Bool` 타입의 클로저를 사용해야 합니다.

클로저를 제공하는 일반적인 방법은 함수 하나를 직접 만들어 인자로 사용하는 것입니다.

```
let numArray: [Int] = [5, 7, 9, 2, 6]

func desc(n1: Int, n2: Int) -> Bool {
    return n1 > n2
}

var sortedArray: [Int] = numArray.sorted(by: desc)
// --> [9, 7, 6, 5, 2]
```
desc 클로저를 `sorted(by:)` 메소드의 인자로 사용하면 배열이 내림차순으로 정렬된 것을 확인할 수 있습니다. 

### 클로저 표현 문법

클로저 표현 문법은 일반적으로 아래의 형태를 하고 있습니다.

```
{ (parameters) -> return type in
    statements
}
```

인자로 사용할 parameters, 처리할 내용을 기술하는 statements, return type 을 가지고 있습니다. 앞서 작성한 desc 클로저를 이용한 sorted 메소드는 다음과 같이 작성할 수 있습니다.

```
sortedArray = numArray.sorted(by: { (n1: Int, n2: Int) -> Bool in
    return n1 > n2
})
```

이렇게 함수로 따로 정의된 형태가 아닌 인자로 들어가 있는 형태의 클로저를 인라인 클로저라고 부릅니다. 예시와 같이 클로저의 body 부분이 짧을 시에는 한줄에 적을 수도 있습니다.

### 문맥에서 타입 추론 & 암시적 반환

`sorted(by:)` 메소드에서 이미 (Int, Int) -> Bool 타입의 인자가 들어와야 하는지 알기 때문에 클로저에서 이 타입들은 생략될 수 있습니다. 

```
sortedArray = numArray.sorted(by: { n1, n2 in return n1 > n2})
```

또한 단일 표현 클로저에서는 `return` 키워드를 생략할 수 있습니다.

```
sortedArray = numArray.sorted(by: { n1, n2 in n1 > n2 })
```

### 인자 이름 축약

Swift 는 인라인 클로저에 자동으로 축약 인자 이름을 제공합니다. 해당 인자를 사용하면 인자 값을 순서대로 $0, $1, $2 등으로 사용할 수 있습니다. 축약 인자 이름을 사용하면 인자 값과 그 인자로 처리할 때 사용하는 인자가 같다는 것을 알기 때문에 인자를 입력받는 부분과 `in` 키워드 부분을 생략할 수 있습니다.

```
sortedArray = numArray.sorted(by: { $0 > $1 })
```

저는 아직 이 방식에 익숙하지 않습니다... Swift 를 더 공부하다 보면 익숙해지겠죠 ㅠㅠ

### 연산자 메소드

여기서 더 간단하게 축약이 가능합니다. Swift 는 타입 연산자끼리 비교 가능한 (Comparable 할 때) 비교 연산자 > 를 구현해 두었습니다. 그렇기 때문에 이 연산자를 바로 사용할 수도 있습니다.

```
sortedArray = numArray.sorted(by: >)
```

## 후위 클로저 (Trailing Closures)

만약 함수의 마지막 인자로 클로저를 넣고 그 클로저가 길다면 후위 클로저를 사용할 수 있습니다.
