## [코딩테스트] 트래블 월렛 - iOS 개발자

오늘 트래블 월렛이라는 회사의 코딩테스트를 치뤘습니다. 

원래는 제가 그래프 순회 문제를 풀이하는데 굉장히 어려움을 느꼈어서 한창 그래프 문제 위주로 학습을 해왔습니다.

그런데 이번 코딩테스트를 준비하며 느낀 점은 DP 부분도 만만치 않게 취약하다는 것이었습니다. 

DP 문제를 몇 문제나 학습하였는데 아직 감도 잡지 못한 것 같습니다. 일단 코딩테스트는 어찌저찌 마무리했지만, 당분간은 DP 문제를 풀어보며 익숙해지는 시간을 가져야겠습니다.

## 코딩테스트

코딩테스트는 210분에 총 3문제로 출제되었습니다. 프로그래머스를 통해 테스트에 응시하였습니다.

### 1번 문제

1번 문제는 주어진 자연수의 각 자릿수를 제곱하여 더하는 것을 반복하여 그 값이 1이 될 때 까지의 횟수를 구하는 것이었습니다.

말로 정리하니까 좀 이상하긴 한데... 문제 자체는 난이도가 높지 않아 간단하게 해결하였습니다.

### 2번 문제

2번 문제는 탐색이었습니다. 2차원 배열 위 0 은 바다, 1은 섬이고 각 섬의 넓이를 구해 중복되는 것을 제외하고 작은 값 부터 배열에 담아 출력하는 문제였습니다.

먼저 배열을 탐색할 때는 BFS 를 이용했습니다. 배열 값이 1 이며 visited 가 false 일때만 탐색했고, 0 이 나오면 BFS 를 종료했습니다.

나오는 넓이들의 값은 Set 에 저장한 후 마지막에 Array 로 변환하여 정렬해주었습니다.

### 3번 문제

가장 어려운 문제였습니다. 배열에서 인접한 값들의 차이가 주어진 값 k 이하가 될 때까지 값들을 swap 해주고 그 횟수를 구하는 문제였습니다.

배열의 최대 크기가 8 이었기 때문에 완전탐색을 이용하려고 했습니다. 

맨 처음에는

1. 가능한 모든 배열의 순서쌍을 구한 후 
2. 인접한 값들의 차이가 k 이하가 되는 배열만 필터링하여
3. swap 횟수가 가장 적은 것을 골라내는 방식

으로 해결할 생각이었습니다.

하지만 3번을 어떤 식으로 해결해야 할지 몰라서 방식을 바꿔보았습니다.

1. 가능한 swap 의 모든 경우의 수를 구하여
2. 그 수가 적은 것 부터 적용시켜보고
3. 인접한 값들의 차이가 k 이하가 되는 순간 리턴

하지만 해당 방식을 사용할 시 3중 반복문을 실행시켜야 했습니다. 그래서 테스트 케이스 8개 중 2개가 시간초과가 발생했는데.. 결국 해결하지 못하고 제출하였습니다 ㅜㅜ

같이 스터디 진행하는 친구가 같은 문제를 풀어봤다고 하여 다음 스터디 시간에 해당 문제에 대해 리뷰해볼 계획입니다.
