## [코딩테스트] 라인 - iOS 개발자

오늘은 라인 코딩테스트를 진행했습니다. 대부분 자잘한 조건이 많이 붙은 구현 문제였고, 제 생각에는 엄청 복잡한 알고리즘은 쓰이지 않은 것 같습니다.

시간에 비해 문제 개수가 많은 편이라 조금 걱정했었는데, 테스트 케이스만 고려했을때에는 6문제 중 5문제를 해결하였습니다.

하지만 테스트 이후 오픈채팅방에서 다른분들과 풀이를 맞춰보고 반례를 적용해보니 모두 완벽하게 해결한 것 같지는 않네요ㅠㅠ

## 코딩테스트

코딩테스트는 3시간에 총 6문제로 출제되었습니다. 프로그래머스를 통해 테스트에 응시하였습니다.

### 1번 문제

1번 문제는 문자열 문제였습니다. 문자열 여러개를 입력으로 받아 주어진 조건들에 부합하지 않는 문자열의 개수를 출력하는 문제였습니다.

연속된 공백의 개수, 문자열에 특수문자나 숫자가 포함되어 있는지 등등... 크게 어렵진 않았지만 자잘한 조건들이 많아 구현하기 쉽지는 않았습니다.

### 2번 문제

2번 문제는 주어진 키보드의 키 개수 n 으로 만들 수 있는 문자열의 길이 중 가장 큰 값을 반환하는 문제였습니다.

문제를 말로 설명하려니까 조금 이상하긴 한데... 여튼 저는 집합, 부분집합을 이용해서 문제를 해결했습니다. 

딕셔너리를 생성하여 key 값으로는 집합을 두고, value 값으로는 문자열의 길이를 두었습니다. key 값들을 모두 비교하며 서로 부분집합일 경우 두 value 를 합치는 방향으로 구현하였습니다.

그런데 테스트가 끝난 후 반례를 대입해봤을때.. 해당 문제는 틀린 것 같습니다. 아마 조합을 이용해서 해결할 수 있을 것으로 보입니다. 

하지만 조합을 이용해서 알파벳의 조합을 구했을 때 시간초과가 나지 않을까 하는 생각도 있습니다. 이 문제는 조금 더 고민이 필요한 것 같습니다.

### 3번 문제

3번 문제도 구현 문제였습니다. 어떤 회사의 팀의 수, 팀마다 팀원 수, 팀원마다의 사원번호, 팀원마다의 업무가 주어지는데 이 중 재택근무를 하는 사원번호를 오름차순으로 출력하는 문제였습니다.

업무는 각각 재택업무, 출근업무로 분류되는데 사실 출근업무가 하나라도 포함되어 있으면 재택을 할 수 없기 때문에 재택업무를 비교할 필요는 없었습니다.

팀의 모든 팀원이 재택근무를 할 시 가장 사원번호가 낮은 사원은 출근해야했기에 팀 번호마다 재택업무를 하게 될 사원을 담은 배열을 지정해준 후, 해당 사원 수가 팀원 수와 같을 시 가장 사원번호가 낮은 팀원을 제외시켰습니다.

### 4번 문제

4번 문제는.. 이렇게 풀어도 되나 싶을정도로 쉽게 구현만 문제였습니다. 셀의 넓이를 담은 배열 두개가 주어지는데 첫번째 배열을 두번째 배열로 만드는데 드는 조절 횟수를 리턴하는 문제였습니다.

셀을 처음부터 탐색하며 i 와 i+1 번째 값을 비교하여 조절해주어 문제를 해결했습니다.

### 5번 문제

5번 문제는 팀원 가르기 문제였습니다. 팀장끼리 번갈아가며 팀원을 고르는데 상대 팀원이 먼저 순서를 가져가고, 저에게는 먼저 순서를 가져갈 수 있는 우선권이 k 장 있습니다.

이 때 가장 팀의 능력치가 높게 할 수 있는 코드를 작성해야 했습니다. 처음에는 그냥 능력이 높은 사람이 나왔을때 우선권을 사용하게 했는데, 팀원간의 능력차를 고려해야 했기 때문에 2명씩 짝지어 능력차대로 정렬한 뒤 능력차가 큰 순으로 우선권을 사용했습니다.

팀원이 짝수인 경우 끝까지 우선권을 한장만 남겨두고 그대로 진행한 뒤 마지막 한장의 우선권을 남은 팀원에게 사용했습니다.

### 6번 문제

6번 문제는... 문제 지문이 너무 길기도 했고 시간이 모자라서 해결하지 못했습니다. 처음 몇 줄 읽었는데 사실 무슨 문제인지 제대로 이해하기도 어려운 것 같아 다른 문제에서 틀린점을 보완하는데 남은 시간을 사용했습니다.

시간이 되면 한번 해결해볼 예정입니다.

## 마무리

테스트 케이스 기준으로는 6문제 중 5문제를 해결했지만 뭔가 찜찜한 기분은 왜일까요... 막상 까보면 전부 틀렸을 수도 있다는 생각이 자꾸 드네요.

이번 코딩테스트는 특별하게 어려운 문제보다 구현하기 까다로운 문제들이 더 많았던 것 같습니다. 그래도 아예 못푸는 문제보다 시간이 좀 걸려도 풀 수 있는 문제가 나은거겠죠?

더 열심히 공부해서 실력을 키우면 제 풀이에 의심을 가지지 않는 날이 올까요? 그럴 수 있을 때까지 진짜 열심히 해야겠다는 생각밖에 들지 않습니다.

앞으로 예정된 코딩테스트는 없지만 이번주에 마감하는 지원서가 많기 때문에 지원서 작성하며 한 주를 보낼 것 같네요. 다음에 또 코딩테스트를 진행하게 되면 후기를 들고 돌아오겠습니다.
